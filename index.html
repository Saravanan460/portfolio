CRISP Data Analysis

View on GitHub

This project is a comprehensive, end-to-end implementation of the Cross-Industry Standard Process for Data Mining (CRISP-DM) methodology. It focuses on analyzing power consumption data from an office building to predict energy usage and identify patterns, demonstrating a structured approach to data science projects.

Project Workflow

  • 1. Problem Understanding

    The initial phase focused on defining the business objective: predicting and optimizing electricity consumption (especially AC units) to reduce costs. The data science goal was to forecast AC power consumption for the next hour based on historical data.

  • 2. Data Collection & Cleaning

    The dataset contained per-minute power consumption and sensor readings for four zones. A major challenge was significant missing sensor data from January to March, which was handled by filtering out this period. Minor gaps were filled using forward/backward fill methods.

  • 3. Feature Engineering

    To help the model understand time-based patterns, new features were created from the timestamp, including hour, day of the week, month, and a boolean for weekends. These temporal features are crucial for predicting cyclical energy usage.

  • 4. Model Training

    An XGBoost Regressor model was chosen for its performance. The model was trained on the engineered features to predict the Zone 1 AC power consumption for the subsequent hour. The data was scaled before training to ensure accuracy.

  • 5. Evaluation & Reporting

    The model's performance was evaluated using RMSE, MAE, and RÂ² Score metrics, achieving an RÂ² of 0.886, indicating a strong predictive capability. Visualizations were created to compare actual vs. predicted values, confirming the model's effectiveness.

Crop Recommendation & Disease Detection App

View on GitHub

An AI-powered mobile application designed for smart farming. It assists farmers by recommending suitable crops based on soil and environmental data, and by detecting plant diseases from leaf images in real-time. The app supports both English and Tamil to serve local communities.

Project Workflow

  • 1. Login/Sign-up

    A secure authentication system allows farmers to create accounts and log in. The interface is bilingual, supporting both Tamil and English to ensure accessibility for local farmers.

  • 2. Crop Recommendation

    Farmers input soil data (N, P, K values), pH, and environmental factors like temperature and rainfall. A trained Random Forest model processes this data to recommend the most suitable crop with 98% accuracy.

  • 3. Disease Detection

    Using the phone's camera, farmers can capture an image of a plant leaf. An EfficientNet B0 model analyzes the image to identify one of 19 plant diseases, providing instant diagnosis with over 90% validation accuracy.

  • 4. Crop Monitoring & Guidance

    After a recommendation or diagnosis, the app integrates with Google's Generative AI (Gemini) to provide detailed guidance on crop duration, water/fertilizer needs, and pesticide recommendations.

  • 5. PDF Report Generation

    The app allows farmers to generate and download timestamped PDF reports for both crop recommendations and disease analysis. This helps in maintaining records for better farm management.

AI Assistant

View on GitHub

A custom, voice-interactive AI companion that runs entirely offline on a local machine. It functions as a digital entity with a distinct personality, voice, and the ability to remember past conversations using a Retrieval-Augmented Generation (RAG) system for long-term memory.

Project Workflow

  • 1. Voice Input Capture

    The system uses the `speech_recognition` library to actively listen for user input through the device's microphone, creating a hands-free interaction experience.

  • 2. Speech-to-Text Conversion

    The captured audio is transcribed into text locally using the `faster-whisper` library, which provides high-accuracy, offline transcription without relying on cloud services.

  • 3. Command Processing (Intent Detection)

    The transcribed text is processed by a local Large Language Model (Llama 3.1). The AI core, configured with a custom personality, analyzes the text to understand the user's intent and consults its ChromaDB vector memory for relevant context from past conversations (RAG).

  • 4. Task Execution

    Based on the detected intent, the assistant formulates a response or prepares to execute a local command. The conversation is logged in a TinyDB database for short-term memory and context.

  • 5. Voice Response Output

    The AI's text response is converted back into natural-sounding speech using the offline Coqui TTS engine. The audio is then played back to the user, completing the conversational loop.

Payir Sei App (TEAM 6 Project)

View on GitHub

A collaborative community farming management platform developed as a team project. The application connects farmers, allows them to list and share resources, track yields and expenses, and generate analytics to improve transparency and efficiency in community farming.

Project Workflow

  • 1. User Registration

    Farmers in a community can register and create profiles, joining a shared digital space. This forms the foundation for resource sharing and collaborative tracking.

  • 2. Resource Listing

    Users can list available resources such as tools, vehicles, or surplus supplies. Other members of the community can view these listings to facilitate borrowing and sharing, optimizing resource utilization.

  • 3. Yield & Expense Tracking

    The app provides an interface for farmers to log their crop yields and associated expenses. This data is stored securely in Firebase, creating a transparent ledger for the community.

  • 4. Report & Analytics Generation

    The platform aggregates the tracked data to generate reports and analytics. This provides insights into the community's overall productivity and financial health, helping with planning and decision-making.
